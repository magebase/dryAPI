# Image-to-Video

> Endpoint for requesting image2video inference

> **NOTE**
>
> **Prerequisite:** To ensure a successful request, you must first consult the [Model Selection](/docs/v1/api/utilities/model-selection) endpoint to identify a valid model `slug`, check specific **limits** and **features**, and verify **LoRA** availability. Please **omit** the LoRA parameter by default during initial testing.

## OpenAPI

````yaml openapi.json post /api/v1/client/img2video

                  type: string

                  type: string
                  format: binary
                last_frame_image:
                  description: >-
                    Image for the last frame of video (optional). Supported

                  type: string

                  type: integer
                  example: 512

                  description: Height of the generated video in pixels
                  type: integer
                  example: 512

                  description: Guidance scale for the generation
                  type: number
                  example: 7.5
                steps:
                  description: Number of inference steps
                  type: integer
                  example: 20
                frames:
                  description: Number of video frames to generate
                  type: integer
                  example: 20
                fps:
                  description: FPS of generated video
                  type: integer
                  example: 30
                seed:
                  description: Random seed for generation
                  type: integer
                  example: 42
                model:
                  description: >-
                    The model to use for video generation. Available models can
                    be retrieved via the GET /api/v1/client/models endpoint.
                  type: string
                  example: Ltxv_13B_0_9_8_Distilled_FP8
                webhook_url:

                    Optional HTTPS URL to receive webhook notifications for job
                    status changes (processing, completed, failed). Must be

                    Documentation](/docs/v1/execution-modes-and-integrations/webhooks)

                  type: string
                  format: uri
                  maxLength: 2048
                  example: https://your-server.com/webhooks/deapi
                  nullable: true
              type: object
      responses:
        '200':
          description: ID of the inference request.
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/JobRequestResponseResource'

          $ref: '#/components/responses/RateLimitExceeded'

        - bearerAuth: []
````
