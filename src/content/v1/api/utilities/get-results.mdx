# Get Results

> Check job status and retrieve results via polling

> **INFO**
>
> **Consider using webhooks or WebSockets instead of polling.**
> 
> Polling requires repeated API calls and adds latency. For production applications, we recommend:
> 
> * **[Webhooks](/docs/v1/execution-modes-and-integrations/webhooks)** — Results pushed to your server automatically (best for backends)
> * **[WebSockets](/docs/v1/execution-modes-and-integrations/websockets)** — Real-time updates with live previews (best for interactive UIs)
> 
> See [Execution Modes](/docs/v1/execution-modes-and-integrations/execution-modes-and-http-queue) for a full comparison.

            application/json:
              schema:
                $ref: '#/components/schemas/JobRequestStatusResource'

          content:

              schema:
                $ref: '#/components/schemas/response_error_default'

            application/json:
              schema:

          $ref: '#/components/responses/RateLimitExceeded'
      security:
        - bearerAuth: []

  parameters:
    AcceptHeader:

      properties:

          properties:

              description: Current status of the job request

                - processing

              nullable: true

                  example: https://example.com/results/image.jpg

                  type: string
                  example: https://example.com/results/image.webp
                  nullable: true
              type: object
              nullable: true

              description: >-
                Generate text (e.g. transcription) Only available when status is

              type: string
              example: https://example.com/results/image.jpg
              nullable: true

              description: Current progress of the job (0.0 to 100.0)

              maximum: 100
              minimum: 0

            type: integer

            type: integer

            type: integer

              - daily
            example: minute
        Retry-After:
          description: >-

          schema:
            type: integer
            example: 60
      content:
        application/json:
          schema:

  securitySchemes:
