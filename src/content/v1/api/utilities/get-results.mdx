# Get Results

> Check job status and retrieve results via polling

<Callout>
**Consider using webhooks or WebSockets instead of polling.**

Polling requires repeated API calls and adds latency. For production applications, we recommend:

* **[Webhooks](/docs/v1/execution-modes-and-integrations/webhooks)** — Results pushed to your server automatically (best for backends)
* **[WebSockets](/docs/v1/execution-modes-and-integrations/websockets)** — Real-time updates with live previews (best for interactive UIs)

See [Execution Modes](/docs/v1/execution-modes-and-integrations/execution-modes-and-http-queue) for a full comparison.
</Callout>
